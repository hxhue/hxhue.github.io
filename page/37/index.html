虚拟继承和虚函数对比

发表于： 2023-08-12 更新于： 2024-05-03 分类于： inside-the-cpp-object-model

目的不同

虚拟继承用来动态访问数据（节约存储空间或保证数据地址唯一），虚函数用来动态访问方法。

虚表的访问方式

如果 A 类有虚函数，那么 A 类已经有 vptr、是多态类，通过 A 类指针/引用访问虚函数则需要查找虚表。

如果 A 类是普通类并且被虚拟继承，那么 vptr 并不会放到 A 类中，通过 A 类指针/引用访问虚函数也不会查表。

考虑 A *pa = new B;，其中 B 类虚拟继承于 A 类，通过 A 类指针 pa 访问到的成员就是真正要找的 A 中的成员，因为从 B 类指针转换到 A 类指针时编译器已经正确处理好了指针偏移问题，从而不需要担心没有 vptr 导致访问不到正确的成员。

用拷贝并交换实现 operator=

发表于： 2023-08-12

问： C++ 中有一种实现赋值运算符的方式：先在栈内用复制/移动构造创建一个新的对象，然后和 *this 交换。不算上标准库或用户实现的 swap 函数的引入，只需要两行代码。请问这种实现方式需要判断赋值运算符的参数指向的对象和 this 相同吗？不判断安全吗？

答： 这种实现方式通常被称为"拷贝并交换（copy-and-swap）"，用于实现赋值运算符。在这种实现方式中，确实不需要判断参数指向的对象和 this 是否相同，因为在执行交换之前，会创建一个参数对象的副本，即使它们相同，也不会产生任何问题。

这种实现方式是安全的，即使参数对象和当前对象是同一个对象。这是因为在执行交换之前，已经创建了参数对象的副本，因此交换操作不会对原始对象产生任何影响。在交换之后，原始对象的资源将被正确释放，而新的副本将具有参数对象的值。

下面是一个简单的示例来说明这个过程：

cppCopy code
class MyClass {
public:
    MyClass& operator=(MyClass other) {
        swap(*this, other); // 交换当前对象和参数对象
        return *this;
    }

    // ... 其他成员函数和数据成员 ...

private:
    // 交换函数
    friend void swap(MyClass& first, MyClass& second) {
        // 交换数据成员
        // ...
    }
};

指向成员的指针

发表于： 2023-08-12 更新于： 2024-04-08 分类于： inside-the-cpp-object-model

参考 https://itanium-cxx-abi.github.io/cxx-abi/abi.html#member-pointers。

向下转换的能力！

和类的指针/引用的转换不同，指向成员的指针既可以向下转，又可以向上转！只需要目的类型拥有对应的成员即可。

struct A    { long a; };
struct B: A { long b; };
int main() {
    // long (A::*p) = &B::b; // 错误，A 类型没有成员 B::b
        long (A::*q) = &B::a; // B::a 和 A::a 是相同的
    long (B::*r) = &A::a; // 向下转
}

指向数据成员

需要存储 this 偏置，按照 Itanium C++ ABI，存储的类型为 ptrdiff_t。一般数据成员指针就用其相对于开头的偏移作为指针的值，而空指针则用 -1 表示。

奇怪的现象：对指向数据成员指针的空初始化会将其置为空指针，也就是 -1！虽然含义为空，但是其比特位并不是全零的。
