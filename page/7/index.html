C++11

C++14

可以省略尾部声明：

#1 相当于 #2。这在 lambda 中是被默认启用的（即便是 C++11 也支持），因而我们写 lambda 经常不必提供返回值类型。例外：但是在构造递归 lambda 时需要显式声明返回值类型，或者先用 function 模板类存储起来，类型也就能从 function 中推导。

C++20

支持给 lambda 声明模板参数。比如：auto f = []<typename T>( T t ) {};

noexcept + throw

发表于 2023-08-12 更新于 2024-03-10 分类于 cpplang
本文字数： 164 阅读时长 ≈ 1 分钟

noexcept 有几个用法：

  1. 在编译期返回一个常量布尔值，评估其表达式是否会抛出异常。
  2. 用于标志一个函数是否能抛出异常，需要一个编译期布尔常量作为参数。
  3. 标志函数不会抛出异常。相当于 noexcept(true)。

C++17 已经禁用 throw 的显式异常声明。（被称为 Dynamic exception specification）

下面几个表达式相等：

  • sizeof(T&)
  • sizeof(T&&)
  • sizeof(T)

也就是说 sizeof 会去掉引用。忘记出处了，twitter 上有人提过一个 quiz，什么类型 T 能满足 struct {T x;} 和 T 的 sizeof 结果不相等，引用类型就满足这样的情况。

对 {} 初始化的理解

可以对基本类型和聚合类零填充。

类型推导分类

见 01 Function Templates。下面讨论的是其中的第 3、4 类。

（嵌套）类型推导

花括号的规则（大概？）是：

  1. 先看对应位置可能出现什么类型，再尝试在花括号前添加这个类型。
  2. B a{3}; 可以看成 B a = {3};
  3. 花括号推导在面临多种推导可能时，会尽可能选择外层和所需类型最接近的一个（例子见下面代码中的 B g2 {{5}}; ）。注意花括号推导并不是函数重载选择，不会因为有多个推导可能而让编译器抱怨 ambiguous。

在 std::vector<int> v {1, 2, 3}; 的情况下，{1, 2, 3} 被认为是 std::initializer_list<int>，所以对 std::vector<int> 初始化。std::vector<int> v {{1, 2, 3}}; 也是等效的，这是其他类型正常初始化的方式（但对容器来说比较冗余）。
