虚拟继承和虚函数对比

发表于 2023-08-12 更新于 2024-03-15 分类于 programming
本文字数： 290 阅读时长 ≈ 1 分钟
  1. 虚拟继承用来动态访问数据（节约存储空间或保证数据地址唯一），虚函数用来动态访问方法。
  2. 如果 A 类有虚函数，那么 A 类已经有 vptr、是多态类，通过 A 类指针/引用访问虚函数则需要查找虚表。
  3. 如果 A 类是普通类并且被虚拟继承，那么 vptr 并不会放到 A 类中，通过 A 类指针/引用访问虚函数也不会查表。

    1. 考虑 A *pa = new B;，其中 B 类虚拟继承于 A 类，通过 A 类指针 pa 访问到的成员就是真正要找的 A 中的成员，因为从 B 类指针转换到 A 类指针时编译器已经正确处理好了指针偏移问题，从而不需要担心没有 vptr 导致访问不到正确的成员。

    2. 考虑菱形继承的情况：

      从指向 D 类真实对象的 B 类指针访问来自 A 中的成员，就会用上 B 类指针所在位置中的 vptr 查找偏置信息。
