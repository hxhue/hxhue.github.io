Docker 配置代理

发表于 2023-12-01 更新于 2024-03-16 分类于 cli
本文字数： 224 阅读时长 ≈ 1 分钟

docker pull

docker pull 不会读取 http_proxy 环境变量，因为负责拉取的是守护进程，环境变量早就在启动时读入，需要专门配置 docker 守护进程并重启它（很不幸，如果和其他人合用同一台服务器，有些服务又在活跃运行，就不能频繁重启了）。可以参考： https://stackoverflow.com/a/71036185

docker build

docker build 的时候应该要用 --build-arg 传代理，然后还得大写：

https://docs.docker.com/network/proxy/#set-proxy-using-the-cli

HTTPS_PROXY 和 HTTP_PROXY 环境变量都得设置。只设置前者是不够的，因为有些连接需要 ssl。

根据 https://stackoverflow.com/a/28093517/ ：

Footnote regarding HTTP_PROXY vs. HTTPS_PROXY: for a long time, setting HTTP_PROXY alone has been good enough. But with version 20.10.8, Docker has moved on to Go 1.16, which changes the semantics of this variable: link

For https:// URLs, the proxy is now determined by the HTTPS_PROXY variable, with no fallback on HTTP_PROXY.
