LLM 在线对话指南

发表于： 2025-04-14 更新于： 2025-05-06
字数： 2337 阅读：≈ 5分钟

叠甲

纯属个人感受，有异议是很正常的。

为什么要被迫窗口切换进行在线对话？因为穷！😢

LLM 擅长回答具体而明确的问题

例子：

  1. 给出两个文件，让 LLM 找出文件的不同。比如给出正确的代码和错误的代码，让 LLM 分析错误的代码哪里错误了。
  2. 联网问一个事实是正确的，还是错误的（fact check）。
  3. 给出文件、修改范例，让 LLM 对文件进行格式修改和内容提取。

我觉得目前的 LLM 还是做题家，能把被训练过的问题做的很好，面对新问题的泛化性稍差一点。

不要把时间浪费在和机器吵架上

如果 few shots1 试了几次 不能正确回答、而且模型也没有指出它需要何种帮助才能更好生成结果，那么说明模型回答不好这样的问题，最好是放弃。

不要事事求证

没有人来夸你懂这懂那的很聪明，不要浪费时间在复制粘贴上。先评估一下有没有较大的把握认为事情是正确的。

是否要联网搜索

联网搜索能提高模型的认识，但搜索到的资料也会影响模型的表现，很多网页工具对搜索到的资料不由分说地认为是正确的，缺乏用交叉检验来判断资料正误的过程。这一点我在 Kimi 和心流上都遇到过，纠正之后它们会承认自己的错误。有些模型的知识库 cut-off 时间已经足够新，问一些不涉及到细节的问题可以不必联网。

常见任务解决方案（截至 2025/4/19）

主要是分为简单任务和复杂任务，简单任务也可能需要专门训练（e.g. OCR），复杂任务需要比较“聪明”的模型，但“聪明”模型却不一定能胜任简单任务（或者说时间 / 价格付出不划算）。

任务类别解决方案备注
代码比对任意思考类模型无
事实检查Grok3 > GeminiGrok3 能直接访问 URL，Gemini 需通过关键词触发搜索
复杂调查报告Gemini Deep Research 或 Grok3 的 Deep(er) Search适用于深入调查报告；Gemini 输出更加正式，如果仅是简单需求建议 Grok3，在 x.com 目前（2025/4/19）是 2 个小时 10 次
Github 代码解读开启联网功能后，使用 Gemini 2.5 Pro 进行搜索需要给出可供搜索的信息
代码 OCRGemini 2.0 Flash纯代码场景较好；普通 OCR 可使用 pixpin、微信，不必使用 LLM
手机 APPGemini 2.0/2.5 Flash免费大碗 + 多模态；中文识别 Gemini > Copilot > Grok3
相似论文比较探索中常忽略论文未强调的信息（但是从论文中可以获取到），输出结果迎合每篇论文的摘要，没有进一步比较
Github 代码仓库学习规划探索中难以实践

💡 2025/4/19 Microsoft Copilot 体验感受是食之无味、弃之可惜：

  1. 输出快但内容很克制，要小心 LLM 时代的 XY 问题；
  2. 可以快速处理文本格式转换等简单问题；
  3. 不像 Grok3 等，中英文搜索效果完全不同！（当成搜索引擎就容易理解了）；
  4. 可笑的是，微软家的搜索效果还不如 Grok3

💡Github 仓库学习规划（🤯探索中），例子：

我刚刚学习大语言模型，发现有个比较好的实践仓库 https://jingyaogong.github.io/minimind 。请你帮我分析这个仓库并列出学习计划，要求详细、分成若干天，每天的时间大概在 6 小时左右，由于模型较小，每次模型训练按照一个 epoch 计算，约 1 小时。假设数据集已经下载。对于模型较为重要的实现细节帮我列举一下，告诉我哪些部分应该去看、去消化。 注意，对于网上出现的资料要判断是否和主题相关，以及严谨程度如何。

作为用户个人偏好的一部分，不应该参考 CSDN 的资料。

不应该参考 CSDN 的资料 是因为心流经常搜索到 CSDN 的资料，但是使用后发现效果更差。让我一下子对心流祛魅了。Gemini 和 Grok 能搜索到海外的资料，在技术领域表现更好。

不过感觉每个模型生成效果都不是很好，虽然说了一大堆，但是：1. 没有和（我）自身实际结合；2. 可实践性不强；3. 任务不够具体。

💡多模态，例子是计算某张截图中餐饮店菜品总价。Gemini 做的比较好，而 Copilot 把菜品中文认错了，Grok 更是把“不用可退”识别成了“不可用”导致少计算了价格。因为国内多模态发展不错，而且中文识别效果好，因此国内大模型在这个场景表现都不错。

常用免费平台优缺点

截至 2025/4/14，我认为免费用户已经完全没必要使用 GPT 了，GPT-4o 对免费用户无论是服务质量还是功能开放上都不够有诚意。以下是我常用的免费平台：

平台优点缺点
Gemini速度超快相比 Google AI Studio 有降智的感觉（个人观点），工具不够灵活，Gemini 2.0 Flash 有时会切换为英文回复
Google AI Studio速度超快，可明确指定是否使用搜索功能或其他工具手机体验较差，包括渲染效果和功能（如缺乏自动保存）。其他缺点与 Gemini 相同
通义在免费平台中多模态功能最令人满意（主要是个人印象好）网页臃肿，思考模式下即使请求联网也不一定联网，服务速度偏慢
心流资料查找方便，有简单模式（一次生成）和深度思考模式（Agent 式处理）有时会找到不可靠的资料导致理解被污染，不会搜索国外的资料
Grok速度快，搜索功能覆盖国内外资料要在 x.com 中使用，难以专注
Deepseek依旧靠谱裸文本模型（搜索、多模态等做得简陋）

总的来说，2025 年 deepseek v3/r1 是预训练 LLM 的守门员。如果和它相比没有优势，那么模型就没有吸引力了。
